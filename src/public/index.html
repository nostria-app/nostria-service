Nostria Service

Comprehensive Backend API for Nostr Applications

About the Service

Nostria Service provides a comprehensive backend API infrastructure for Nostr applications. It offers user management, subscription services, account operations, secure authentication, and administrative tools. The service is designed to support Nostr clients with scalable, reliable backend functionality using NIP-98 authentication for security.

Features

Complete user registration, profile management, and account operations for Nostr applications.

🔒

NIP-98 Authentication

Secure authentication using Nostr signatures with NIP-98 HTTP authentication protocol.

API Endpoints

The following endpoints are available for developers:

Method Endpoint Description
GET /api/status Get service information and status
GET /api/status/health Health check endpoint
GET /api/health Comprehensive health monitoring and system status
POST /api/signup/register Register a new user account
GET /api/signup/profile/:pubkey Get user profile information
GET /api/account/profile Get authenticated user's account profile
PUT /api/account/profile Update authenticated user's account profile
GET /api/subscription-management/pricing Get subscription pricing information
GET /api/subscription-management/status Get current subscription status (authenticated)
POST /api/subscription-management/upgrade Upgrade to premium subscription
POST /api/subscription-management/cancel Cancel current subscription
PUT /api/subscription-management/auto-renew Update auto-renewal settings
GET /api/admin/users Get user management dashboard (admin only)
GET /api/admin/analytics Get system analytics and metrics (admin only)

Note: Most API endpoints require NIP-98 HTTP Authentication. Admin endpoints require elevated privileges.

Authentication

This API uses NIP-98 HTTP Authentication for secure access to endpoints:

  • Some API requests must include authentication headers as specified in NIP-98
  • You need to sign your requests with your Nostr private key
  • Include the following headers:
    • Authorization: Nostr [base64-encoded-event]
    • The event must include the HTTP method, path, and optional request body
  • The server verifies the signature against your public key to authenticate the request

For more details on implementation, see the NIP-98 Specification.

Getting Started

To use the Nostria Service, you need to:

  1. Register a user account using the signup API
  2. Authenticate requests using NIP-98 HTTP Authentication with your Nostr keys
  3. Configure your account profile and subscription preferences
  4. Integrate with the API endpoints for your Nostr application needs

For developers who want to integrate with the service, ensure your application supports NIP-98 authentication. Admin access is available for authorized users with elevated privileges.
